## Spring Bean

**정의**: Spring 컨테이너가 **관리하는 객체**

### 사용 이유

- **싱글톤 보장**
- **의존성 주입**: Spring이 알아서 주입해줌
- **생명주기 관리**

### 스프링 탐색 과정

1. **타입 검색**
- 일치하는 타입이 있는지, 최우선으로 검색
1. **개수 확인**
- **Case A: 후보가 1개**
    - 해당 빈 전달
- **Case B: 후보가 2개 이상**
    - **name 확인**
        - **일치**: 해당 빈 전달
        - **불일치**: 에러 발생

<br>

## @Configuration

**정의**: **Spring Bean 설정 정보**가 들어있는 파일이라는 것을 명시하는 어노테이션

### 특징

- 해당 어노테이션이 붙은 클래스는, Spring이 설정 파일로 인식

### 사용 이유

- Spring 컨테이너에게 ‘어떤 객체를 생성하고, 그 객체들 간의 관계는 어떤지’ 알려주기 위해
- **CGLIB(Code Generator Library)**: @Configuration 안에서 @Bean 메서드를 여러 번 호출해도, 
실제로는 싱글톤이 반환되도록 함

<br>

## @Bean

**정의**: 개발자가 정의한 **메서드의** **반환 값**을 Spring Bean으로 등록하는 어노테이션

### 작동 원리

1. Spring이 시작될 때, @Configuration 클래스를 읽음
2. @Bean이 붙은 메서드 실행
3. 메서드 반환 값을 Spring 컨테이너에 저장
4. 이때 Bean의 기본 이름은 메서드 명(어노테이션에서 `name = " "`로 설정 가능)

### 사용 이유

- 외부 라이브러리 클래스(e.g. ObjectMapper, DataSource 등)를 Bean으로 등록하고 싶을 때 
주로 사용
→ 외부 코드는 마음대로 @Component를 못 붙이기 때문에

<br>

## AnnotationConfigApplicationContext

**정의**: @Configuration과 @Bean 같은 어노테이션 기반의 설정을 읽어들여서, 실제 Bean 생성 및 
관리하는 **스프링 컨테이너**

### 역할

- ApplicationContext 인터페이스 구현체 중 하나

<br>

## record

**정의**: 데이터를 담기 위한 불변 객체를 간결하게 만들기 위한 문법

### 특징

- **불변성**: record로 만든 객체의 모든 필드는 자동으로 `private final`. ****
- **보일러플레이트(Boilerplate) 제거**: `toString()`, `equals()`, `hasCode()`, `Getter` 메서드 컴파일러가 자동 생성
- **간결함**
