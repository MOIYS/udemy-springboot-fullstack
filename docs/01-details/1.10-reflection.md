## 리플렉션(Reflection)

**정의**: 구체적인 클래스 타입을 몰라도, runtime에 해당 클래스의 정보(메서드, 필드, 부모 클래스 등)을 알아내거나 조작할 수 있는 Java가 제공하는 기능  

**역할**

1\. **정보 조회:** 클래스명, 메서드, 필드, 부모 클래스, 어노테이션 등을 알 수 있음
<br>
2\. **제어 조작**: 캡슐화를 깨뜨릴 수 있음 
- `private` **메서드 강제 호출**
- `private` **필드 값 변경**
    - 필드 주입이 가능한 이유

3\. **동적 객체 생성**: `new` 키워드 없이, 클래스 이름으로 객체 생성 가능

**스프링에서 사용하는 방식**

- Java가 제공하는 리플렉션 기능을 활용하여 DI 컨테이너를 만듦

**주의사항**

1/. **속도 저하**: runtime에 하나하나 분석하여 처리하니 느림
                   (스프링은 시작할 때 한 번만 수행)
<br>
2/. **안정성 부족**: 컴파일 에러를 잡을 수 없음 → (에러가 발생하면) runtime error
<br> 
3/. **객체지향 파괴**: `private` 같은 접근 제어자를 무시하기 때문에, 내부 설계를 망칠 수 있음
