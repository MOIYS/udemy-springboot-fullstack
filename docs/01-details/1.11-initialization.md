## Eager Initialization


**정의**: **애플리케이션이 실행되는 시점**에 관련된 모든 빈을 생성하고 연결을 마치는 초기화

**특징**

- **Fail Fast (빠른 실패)**: 애플리케이션 **시작 시점에 에러 발견** 가능
- **메모리 점유**: 당장 안 쓰는 Bean이 있더라도, **모두 생성**하니 메모리를 차지함
- **빠른 첫 요청**: 이미 만들어져 있으니, 지체 없이 실행 가능
- **스프링 기본값(대부분 해당 방식 사용)**

<br>

## Lazy Initialization


**정의**:  (애플리케이션 시작 시점에 Bean을 만들지 않고) 그 **Bean을 필요로하는 시점(첫 요청)**에 생성하는 초기화 ****

**특징**

- **빠른 실행**: (시작할 때) Eager보다 Bean을 덜 만들어, 애플리케이션 **부팅 속도가 빠름**
- **메모리 절약**: 사용하지 않는 Bean은 메모리에 올리지 않음
- **첫 요청 느림**: 처음 사용할 때, **생성 로직으로 인해** 약간의 지연 발생 가능
- **런타임 에러**: 에러가 있어도, 앱 실행 시에는 모르고 **실행할 때 발견 가능**

<br>

## @Lazy


**정의**: Lazy Initialization 개념을 구현하는 어노테이션

**역할**

1. **빈 생성 지연 (Component/Bean 레벨)**
- **상황**: `@Component`나 `@Bean` 위에 붙였을 때
- **역할**: 앱 시작 시, 해당 Bean 생성을 건너 띔. 나중에 요청 시 생성

1. **의존성 주입 지연(Injection 레벨)**
- **상황**: **생성자**나 `@Autowired` 필드에 붙였을 때
- **역할**: 가짜 객체(Porxy)를 주입 해둠. 실제로 메서드 호출 시 생성 및 연결
- **사용처**: 순환 참조 문제 해결에 사용

**주의점**

```java
@Component
public class EagerBoss {
	
	private final LazyEmployee employee;  // Lazy 빈 의존
	
	// Eager 빈 생성 시, Lazy 빈 필요 -> Lazy 빈도 즉시 생성
	public EagerBoss(LazyEmployee employee) {
			this.employee = employee;
	}
}

@Component
@Lazy
public class LazyEmployee {...}
```

- **상황**: Eager 빈이 Lazy 빈을 의존할 때
- **결과**: `@Lazy`가 붙어있어도, Eager 빈이 생성될 때 Lazy 빈도 즉시 생성
- **해결책(Lazy하게 하고 싶으면)**: 주입 받는 곳에도 `@Lazy`를 붙임(프록시 객체가 들어감)
```java
public EagerBoss(@Lazy LazyEmployee employee) {...}
```
