## Jakarta EE

**정의**: 자바를 이용한 서버측 개발을 위한 기술들의 표준 사양 모음

**주요 기술**

- **JSP(Java Server Pages)**: 동적 웹페이지를 만드는 기술
- **JSTL(JSP Standard Tag Library)**: JSP에서 태그 형태로 로직을 짤 수 있게 도와주는 
라이브러리
- **EJB(Enterprise Java Beans)**: 기업 환경의 시스템을 구현하기 위한 서버 측 컴포넌트 
모델 (e.g. 트랜잭션, 보안 등)
    - → 복잡하고 무거움 ⇒ Spring Framework 탄생 계기
- **JAX-RS(Java API for RESTful Web Services)**: REST API를 만들기 위한 표준 규격 (e.g. `@GET`, `@Path` 등
- **Bean Validation**: 데이터의 유효성을 검사하는 표준 (e.g. `@NotNull`, `@Size` 등)
- **CDI(Contexts and Dependency Injection)**: 자바 표준 DI 기술
- **JPA(Java Persistence API)**: 자바 객체와 DB를 매핑해주는 ORM 표준 인터페이스

**Jakarta EE 발전 과정**

- J2EE → Java EE → Jakarta EE


> [!TIP]
> Spring Framework 6 & Spring Boot 3부터 Jakarta EE 지원 
(javax가 아니라 jakarta 사용)
