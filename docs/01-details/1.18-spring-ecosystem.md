# Spring Framework

### 1. Core (IoC Container, Dependency Injection, Auto Wiring etc)

**정의**: Spring의 기초가 되는 부분

**특징**

- 객체의 라이프사이클 관리

### 2. Web (Spring MVC etc)

**정의**: 동기 방식의 웹 애플리케이션을 만드는 모듈

**특징**

- **Servlet 기반**: Servlet API 위에서 동작
- **Spring MVC**: MVC 구조를 제공하여 웹 요청 처리 ****

### 3. Web Reactive (Spring WebFlux etc)

**정의**: 비동기와 논블로킹 방식을 지원하는 웹 모듈

**특징**

- **이벤트 루프 방식**: 적은 수의 스레드로 많은 동시 요청 처리 가능
- **고성능**: 대용량 트래픽 처리에 사용



### 4. Data Access (JDBC, JPA etc)

**정의**: DB 통신을 쉽게 해주는 모듈

**특징**

- **JDBC**: 자바에서 DB에 접속할 수 있게 하는 자바 API
- **Transaction**: `@Transactional` 어노테이션으로 트랜잭션 관리를 자동으로 해줌
- **ORM(JPA 등)**: SQL 없이 자바 객체로 DB를 다루는 기술

### 5. Integration: JMS etc

**정의**: 자신이 속한 서버가 아닌, 다른 시스템과 소통하기 위한 도구들

**특징**

- **JMS(Java Message Service)**: 메시지 큐(Kafka, RabbitMQ 등)와 통신하여 데이터를 주고 받음
- **Email, Scheduling**: 이메일 기능, 스케줄링 기능 등 포함

### 6. Testing: Mock Objects, Spring MVC Test etc

**정의**: 테스트 검증 도구

**특징**

- **Mock Objects**: 가짜 객체(Mock)를 쉽게 만들어 테스트 가능
- **Spirng MVC Test**: 서버를 실제로 켜지 않고도 URL 요청 테스트를 할 수 있게 해줌

# Spring Projects


### 1. Spring Framework

**역할**: 핵심 컨테이너, 트랜잭션 관리, 웹앱, 데이터 접근 등 제공

### 2. Spring Boot

**역할**: 스프링 기반 애플리케이션을 빠르고 쉽게 실행할 수 있는 상태로 만들어줌

### 3. Spring Security

**역할**:  인증(Authentication)과 인가(Authorization) 담당. 보인 공격 방어 기능 제공

### 4. Spring Data

**역할**: 여러 DB에 접근하는 방식을 통일된 인터페이스로 제공. 반복적인 CRUD 코드도 줄여줌 

### 5. Spring Integration

**역할**: 애플리케이션 내의 서로 다른 시스템 간의 데이터 흐름(메시징, 파일 처리, FTP 등) 연결

### 6. Spring Cloud

**역할**:  마이크로서비스 아키텍처(MSA) 구축에 필요한 분산 시스템 패턴을 제공
