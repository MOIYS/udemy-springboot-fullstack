## Spring Container란?

**정의**: Spring Bean의 lifecycle을 관리하는 컴포넌트 

- **Spring Container = Spring Context = IOC Container**
    - 엄밀히 따지면 동일이 아닌 포함 관계에 가깝지만, 현재 학습 중에는 동일하다 보고 
    넘어가도 무방

<br>

## Bean Factory / Application Context

### Bean Factory

**정의**: 기본 스프링 컨테이너

**특징**

- **지연 로딩(Lazy Loading)** - 빈이 요청될 때 생성

### Application Context

**정의**: BeanFactory 확장 컨테이너


**특징**

- BeanFactory 모든 기능 포함 + 국제화 + 이벤트 전파 등 추가 기능 제공
- **즉시 로딩(Eager Loading) -** 애플리케이션 시작 시, 모든 빈을 미리 로딩
- 특별한 이유 없다면 권장되는 컨테이너
