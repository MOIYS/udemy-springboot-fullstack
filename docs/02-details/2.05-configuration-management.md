# profile을 사용하여 애플리케이션 설정 관리

### profile

**정의**: 개발(dev), 운영(prod) 등 환경 분리 역할. 환경에 따라 서로 다른 설정값을 로딩하게 해줌

<br>

**동작 원리**

- **기본 설정**: `application.yaml`
- **환경별 설정**: `application-{profile}.yaml` 형식
    - e.g. `application-**dev**.yaml`, `application-**prod**.yaml`

<br>

### Logging 레벨

- `trace` -> `debug` -> `info` → `warning` → `error` → `off`
    - 오른쪽으로 갈수록 심각도가 높음
    - 설정된 레벨 이상의 로그만 출력
        - e.g. `info`로 설정 → `info`, `warning`, `error`가 출력 됨

<br>

**레벨**

- **TRACE**:
    - **정의**: 가장 상세한 정보. 애플리케이션 실행 흐름과 디버깅 정보
    - **내용**: 메서드 진입/종료, 루프 각 단계, 변수 값의 순간순간 변화 등 기록
- **DEBUG**:
    - **정의**: 시스템 내부 상태와 흐름 정보
    - **내용**: 데이터가 어떻게 변형되고 있는지, 어떤 분기를 탔는지
- **INFO**:
    - **정의**: 애플리케이션의 주요 진행 상황 정보
    - **내용**: ‘~~ 프로세스가 완료됐다’, ‘서버가 8080 포트에서 시작 됐다’ 등의 상태 요약 데이터
- **WARN**:
    - **정의**: 잠재적 위협 정보
    - **내용**: ‘디스크 용량이 부족해지고 있음`, ‘DB 응답이 평소보다 느림’ 등의 경고성 데이터
- **ERROR**:
    - **정의**: 기능 실패 정보
    - **내용**: ‘NullPointerException’, ‘DB 연결 시간 초과’ 등의 예외 발생 데이터
- **OFF**: 로그를 남기지 않음

<br>

# @ConfigurationProperties


**정의**: 설정 파일의 값을 자바 객체로 안전하게 매핑해주는 기술

<br>

**특징**

**1. 타입 안정성(Type Safety)**
- 설정 파일의 문자열을 `Integer`, `Boolean`, `List`, `Map` 등 자바 타입으로 자동 변환

<br>

**2. 관련 설정 그룹화**
- DB 설정 등 연관된 설정값들을 하나의 클래스로 묶어서 관리 가능
