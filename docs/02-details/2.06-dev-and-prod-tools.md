# Spring Boot DevTools

**정의**: 개발 단계의 생산성 향상을 위한 모듈

<br>

### **핵심 기능**

**1. 자동 재시작 (Automatic Restart)**

- **기능**: 코드가 수정되면 자동으로 서버를 껐다 켜줌
- **원리**:
    - **Base ClassLoader**: 라이브러리(JAR) 등 바뀌지 않는 것들을 미리 로딩해둠
    - **Restart ClassLoader**: 개발자가 작성 중인 코드만 로딩
    - 코드 수정 시, **Restart ClassLoader**만 버리고 새로 만듦. 그래서 일반적인 재시작보다 속도가 빠름

<br>

**2. 브라우저 자동 새로고침 (LiveReload)**
- **기능**: 정적 자원(HTML, CSS, JS 등)을 수정하면, 브라우저를 **자동으로 새로고침**

<br>

**3. 탬플릿 캐시 비활성화 (Property Defaults)**

- **기능**: 템플릿 캐시를 **자동으로 꺼줌**

> [!TIP]
> - **템플릿 캐시(Template Cache)**: 서버가 매번 파일을 새로 읽지 않도록, 한 번 해석한 내용을 메모리에 기억해두는 것
> - pom.xml `dependency` 변경의 경우에는, 서버를 수동으로 재시작해야 함


<br>

# Actuator


**정의**: 운영 환경에서 애플리케이션 모니터링 및 관리 기능을 제공하는 라이브러리

<br>

### 핵심 엔드포인트

**1.health:** 서버가 살아있는지(UP/DOWN), DB 연결이 잘 되어있는지 등을 보여줌

**2. beans:** 스프링 컨테이너에 등록된 모든 빈 목록

**3. metrics**: HTTP 요청 횟수, CPU 사용량, 메모리 사용량 등의 데이터 

**4. mappings**: `@RequestMapping`으로 연결된 URL 목록
