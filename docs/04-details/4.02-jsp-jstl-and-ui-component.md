## JSP(Java Server Pages)

**정의**: HTML 코드 안에 자바 코드를 삽입하여 웹 서버에서 동적으로 웹 페이지를 생성하는 서버 사이드 스크립트 언어

<br>

**특징**

- **서버 사이드 렌더링(SSR, Server Side Rendering)**: 브라우저가 아닌, 서버에서 HTML을 완성한 뒤 클라이언트에 보냄
- **서블릿 변환**: JSP 파일은 실행 시 내부적으로 **서블릿(.java) 소스 코드로 변환** 되고, 컴파일 과정(.class)을 거쳐 실행 ****
- **tomcat-embed-jasper 의존성 필수**: 스프링 부트 내장 톰캣은 기본적으로 JSP 파싱 엔진을 포함하지 않음. JSP를 컴파일 하고 실행하기 위해서는 `tomcat-embed-jasper` 의존성 추가 필수

<br>

### JAR vs WAR 패키징 구조 차이

1. **JAR 패키징 (스프링 부트 기본)**
- 스프링 부트는 내장 톰캣을 포함한 단일 JAR 파일로 배포되는 것이 표준
- JAR 내부의 `/META-INF/resources` 폴더가 웹 애플리케이션 루트(`/`)로 매핑됨
- **표준 경로**: `src/main/resources/META-INF/resources/WEB-INF/jsp/`

<br>

2. **WAR 패키징 (레거시/외장 톰캣)**
- 전통적인 웹 애플리케이션 구조
- **표준 경로**: `src/main/webapp/WEB-INF/jsp/`

<br>

### ViewResolver 설정 (application.yaml)

- **mvc.view.prefix**: `/WEB-INF/jsp/` (뷰 이름 앞에 붙일 경로)
- **mvc.view.suffix**: `.jsp` (뷰 이름 뒤에 붙일 확장자)

<br>

**동작 원리**

- e.g. 컨트롤러 리턴값: `"hello"`
- 최종 경로 조합: `/WEB-INF/jsp/` + `"hello"` + `.jsp`
- 파일 탐색 위치: `src/main/resources/META-INF/resources/WEB-INF/jsp/hello.jsp`

<br>
<br>

## JSTL (JSP Standard Tag Library)

**정의**: JSP 페이지 내에서 자바 코드를 사용하지 않고, 태그 형태로 로직을 처리할 수 있도록 지원하는 표준 태그 라이브러리 모음

<br>

**특징**

- **가독성 향상**
- **유지보수 용이**
- **EL과 연동**

<br>

### 주요 태그 라이브러리

1. **Core (`c`):** 가장 많이 사용되는 기능 모음
- **변수 지원**
    - `<c:set>`
    - `<c:remove>`
- **흐름 제어**
    - `<c:if>`
- **반복문**
    - `<c:forEach>`

<br>

2. **Formatting (`fmt`):** 날짜, 숫자 형식을 국가별/패턴별로 출력
- e.g. `<fmt:formDate value="${date}" pattern="yyyy-MM-dd" />`

<br>

3. **Functions (`fn`)**: 문자열 처리 함수들 제공
- e.g. `${fn:length(list)}`

<br>
<br>

## Bootstrap

**정의**: HTML, CSS, JavaScript로 구성된 오픈소스 프론트엔드 프레임워크

<br>

**기능**

- **미리 정의된 스타일:** 버튼, 입력 폼, 테이블, 경고창 등 클래스 이름만 지정하면 미리 정의된 스타일 적용 가능
- **유틸리티 클래스**: 마진, 패딩, 텍스트 정렬 등을 CSS 파일 수정 없이 HTML 클래스만으로 적용 가능

**특징**

- **반응형 웹 디자인(Responsive Web Design):** 화면 크기에 맞춰 **레이아웃이 자동으로 조절됨**
- **그리드 시스템(Grid System)**: **화면을 12개의 컬럼**으로 나누어 배치할 수 있는 레이아웃 시스템 제공
- **WebJars 활용**: 스프링 부트 환경에서 CSS/JS 파일을 직접 다운로드하지 않고, **의존성으로 관리** 
가능

<br>

### bootstrap-datepicker

**정의**: Bootstrap 프레임워크 기반으로 동작하는 날짜 선택 플러그인

<br>

**기능**

- **시각적 선택**: 사용자가 달력에서 클릭하여 날짜를 입력할 수 있게 함
- **자동 포맷팅**: 사용자가 선택한 날짜를 지정된 문자열 포맷으로 변환하여 입력 필드에 채워줌

**특징**

- **의존성**: jQuery와 Bootstrap JS가 먼저 로드되어야 동작
- **설정 가능**: 날짜 형식, 언어 설정, 선택 가능 날짜 범위 등 옵션들을 자바스크립트로 제어 가능
- **데이터 정합성 보장**: 서버가 기대하는 포맷(e.g. `yyyy-MM-dd`)으로 데이터를 전송하도록 강제함

<br>
<br>

## JSP Fragment (`.jspf`)

**정의**: JSP 페이지 내에서 공통적으로 반복되는 부분을 별도의 JSP 파일로 분리한 조각

<br>

**특징**

- **모듈화**
- **DRY 원칙 (Don’t Repeat Yourself)**: 똑같은 코드를 여러 페이지에 반복하지 않음
- **유지보수 효율성 증대**
- **레이아웃 일관성 유지**
